## 1. 이미지 리사이징(Resizing)과 정규화(Normalization)를 하는 이유

### 팩터 1: 이미지 리사이징 (Resizing)

* **연산의 일관성 (Fixed Input Size):** CNN(합성곱 신경망)의 완전연결층(Fully Connected Layer)이나 최신 비전 트랜스포머(ViT) 모델은 입력 데이터의 크기가 고정되어 있어야 합니다. 다양한 크기의 이미지를 동일한 크기로 맞춰야 하나의 배치(Batch)로 묶어 GPU 연산을 효율적으로 수행할 수 있습니다.
* **컴퓨팅 자원 최적화:** 너무 큰 원본 이미지는 대규모 파라미터를 가진 딥러닝 모델의 연산 속도를 저하시키고 OOM(Out of Memory) 오류를 유발합니다. 따라서 모델이 학습하기 적절한 해상도(예: 224x224, 384x384 등)로 축소하여 연산 효율을 높입니다.

### 팩터 2: 이미지 정규화 (Normalization)

* **경사하강법(Gradient Descent)의 안정성 및 수렴 속도 향상:** 이미지의 픽셀 값은 기본적으로 0~255 사이의 정수입니다. 이를 정규화하지 않고 그대로 입력하면 입력 값의 스케일이 커서 그래디언트가 폭발하거나 손실 함수(Loss Landscape)가 왜곡되어 학습이 불안정해집니다.
* **0~1 또는 -1~1 스케일링:** 픽셀 값을 255로 나누어 0~1 사이로 맞추거나, 데이터셋 전체의 평균(Mean)과 표준편차(Std)를 활용해 평균 0, 분산 1을 갖는 표준정규분포 형태로 변형합니다. 이렇게 하면 활성화 함수(Activation Function)가 활성 영역에서 효과적으로 동작하며, 최적의 가중치(Weight)를 찾는 속도가 훨씬 빨라집니다.

## 2. 데이터 증강(Data Augmentation)의 정의와 주요 기법

### 데이터 증강이란?

훈련 데이터셋의 양이 적을 때, **기존의 이미지를 인위적으로 변형시켜 데이터의 다양성을 확보하는 기법**입니다. 모델이 훈련 데이터에만 과적합(Overfitting)되는 것을 방지하고, 현실 세계의 다양한 변수(조명, 각도 등)에 대응할 수 있도록 일반화 성능(Generalization)을 극대화하는 역할을 합니다.

### 이미지 데이터에서 주로 사용하는 증강 기법

* **기하학적 변형 (Geometric Transformations)**
* **Flip (뒤집기):** 좌우 반전(Horizontal) 또는 상하 반전(Vertical). (단, 숫자나 글자 인식 모델에서는 상하좌우 반전 시 의미가 변하므로 주의해야 합니다.)
* **Rotation (회전) & Translation (이동):** 이미지를 일정 각도만큼 회전하거나 상하좌우로 밀어서 사물이 중심에 있지 않은 상황을 학습시킵니다.
* **Crop & Resize (자르기 및 크기 조정):** 이미지의 특정 부분만 무작위로 잘라내어 확대한 후 모델에 입력합니다. 사물의 일부만 보여도 인식할 수 있도록 돕습니다.


* **픽셀/색상 변형 (Color Space Transformations)**
* **Color Jitter:** 이미지의 밝기(Brightness), 대비(Contrast), 채도(Saturation), 색조(Hue)를 임의로 변경합니다. 촬영 환경의 조명 변화에 강건한 모델을 만듭니다.


* **고급/혼합 변형 (Advanced Regularization)**
* **Random Erasing / Cutout:** 이미지의 일부 영역을 무작위로 사각형 모양으로 지우거나(0 또는 평균값으로 채움) 가립니다. 모델이 특정 국소적 특징(예: 새의 부리만 보고 새라고 판단하는 것)에만 의존하지 않도록 강제합니다.
* **Mixup & CutMix:** 두 개의 이미지를 일정 비율로 합성하거나(Mixup), 한 이미지의 일부를 잘라내어 다른 이미지에 붙여넣는(CutMix) 방식입니다. 레이블 역시 비율에 맞춰 혼합하므로 모델의 결정 경계(Decision Boundary)를 부드럽게 만들어 일반화 성능을 크게 높입니다.

## 3. 전이 학습(Transfer Learning)의 정의와 이미지 분류 활용법

### 전이 학습이란?

수많은 데이터(예: 수백만 장의 ImageNet)로 **이미 대규모 데이터셋에서 학습을 완료한 사전 학습 모델(Pre-trained Model)의 지식(가중치)을 가져와, 내가 해결하고자 하는 유사한 목적의 새로운 작업(Target Task)에 재사용하는 기법**입니다.

데이터가 부족하거나 컴퓨팅 자원이 한정된 상황에서 처음부터(Scratch) 모델을 학습시키는 것보다 훨씬 높은 정확도와 빠른 수렴 속도를 보장합니다.

### 이미지 분류 모델에서의 구체적인 활용 방법

사전 학습된 모델(예: ResNet, EfficientNet 등)은 대개 앞단에서 선, 면, 질감 같은 일반적인 특징(General Features)을 추출하고, 뒷단의 분류기(Classifier)에서 구체적인 클래스를 분류합니다. 이를 활용하는 방법은 크게 두 가지로 나뉩니다.

* **방법 1: 특성 추출기 (Feature Extractor)로 활용**
* **설명:** 사전 학습된 모델의 Backbone(합성곱 층) 가중치를 고정(Freeze)하여 업데이트되지 않도록 하고, 맨 마지막의 출력층(Fully Connected Layer/Classification Head)만 내 데이터셋의 클래스 개수에 맞게 새로 설계하여 해당 부분만 학습시킵니다.
* **언제 쓰나요?:** 내 데이터셋의 크기가 **매우 작고**, 사전 학습된 데이터셋과 **시각적 유사성이 높을 때** 유용합니다. 기존 모델의 훌륭한 특징 추출 능력을 그대로 보존하면서 빠르게 분류기만 학습할 수 있습니다.


* **방법 2: 미세 조정 (Fine-tuning)**
* **설명:** 분류기만 새로 학습하는 것에 그치지 않고, 가중치가 얼어있던 Backbone의 일부 상위 레이어 또는 전체 레이어의 봉인을 해제(Unfreeze)하여 내 데이터셋에 맞게 가중치를 미세하게 다시 학습(Update)시키는 방법입니다. 이때 급격한 가중치 변화로 기존 지식이 파괴되는 것을 막기 위해 보통 매우 낮은 학습률(Learning Rate)을 설정합니다.
* **언제 쓰나요?:** 내 데이터셋의 크기가 **어느 정도 충분히 확보되었거나**, 사전 학습된 데이터셋과 **시각적 특성이 많이 다를 때** 사용합니다. 모델이 새로운 데이터셋의 도메인 특성에 맞게 완전히 동화되도록 만듭니다.
