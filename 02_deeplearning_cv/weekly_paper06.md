## 1. CNN(Convolutional Neural Network)을 구성하는 레이어별 역할

CNN은 주로 이미지의 공간적 정보(Spatial Information)를 유지하며 유기적인 특징을 추출하기 위해 사용되는 신경망입니다. 각 레이어는 다음과 같은 고유한 역할을 수행합니다.

### 📊 입력 레이어 (Input Layer)
* **역할:** 이미지의 원본 픽셀 데이터를 그대로 받아들이는 채널입니다. 
* **특징:** 일반적으로 `[가로, 세로, 채널 수(RGB=3 또는 흑백=1)]` 형태의 3차원 텐서 구조로 데이터를 입력받습니다.

### 🔍 합성곱 레이어 (Convolutional Layer)
* **역할:** 이미지의 **국소적 특징(Local Features)을 추출**하는 핵심 레이어입니다.
* **원리:** 이미지 위를 일정한 간격(Stride)으로 이동하는 **필터(Filter 또는 Kernel)**를 활용하여 합성곱 연산을 수행합니다. 필터는 이미지의 선, 면, 질감 등의 특징을 찾아내며, 그 결과물로 **특징 맵(Feature Map)**을 생성합니다.

### ⚡ 활성화 함수 레이어 (Activation Function Layer - 주로 ReLU)
* **역할:** 신경망에 **비선형성(Non-linearity)**을 부여합니다.
* **원리:** 합성곱 연산은 선형 결합이므로, 이를 그대로 쌓으면 단층 레이어와 다름없어집니다. `ReLU` 함수 등을 적용하여 음수 값은 0으로 바꾸고 양수 값은 그대로 통과시킴으로써 복잡한 패턴을 학습할 수 있는 기반을 만듭니다.

### 📉 풀링 레이어 (Pooling Layer - 주로 Max Pooling)
* **역할:** 특징 맵의 공간적 크기를 줄여 **컴퓨팅 연산량을 감소시키고 오버피팅(과적합)을 방지**합니다.
* **원리:** 특정 영역 내에서 가장 큰 값을 추출하는 **맥스 풀링(Max Pooling)**이 주로 사용됩니다. 이 과정에서 이미지 내 특징의 위치가 미세하게 변하더라도 공간적 불변성(Translation Invariance)을 확보할 수 있게 됩니다.

### 🔗 전결합 레이어 (Fully Connected Layer / FC Layer)
* **역할:** 추출된 고차원 특징들을 하나로 모아 **최종적인 분류(Classification) 또는 회귀(Regression) 예측**을 수행합니다.
* **원리:** 3차원 형태의 특징 맵을 1차원 벡터로 평탄화(Flatten)한 후, 일반적인 다층 퍼셉트론(MLP) 형태로 모든 뉴런을 연결하여 최종 점수를 계산합니다.

## 2. 오토인코더(Autoencoder)의 활용 및 구성 요소

오토인코더는 입력 데이터 자체를 레이블(정답)로 사용하는 대표적인 **비지도 학습(Unsupervised Learning)** 신경망입니다.

### 💡 오토인코더가 적용되기 적합한 상황

* **이상 탐지 (Anomaly Detection):** 정상 데이터만을 학습한 오토인코더는 정상적인 패턴을 잘 복원하지만, 이상 데이터가 입력되면 복원 성능이 떨어집니다. 이때 발생하는 **복원 오차(Reconstruction Error)의 크기를 기준**으로 불량품이나 이상 징후를 감지할 수 있습니다.
* **노이즈 제거 (Denoising):** 입력 데이터에 고의로 노이즈를 섞은 후, 원본의 깨끗한 이미지를 복원하도록 학습(Denoising Autoencoder)시키면 이미지나 오디오의 잡음을 효과적으로 제거할 수 있습니다.
* **차원 축소 및 특징 추출 (Dimensionality Reduction):** 고차원의 복잡한 데이터를 중요한 정보만 남긴 고유의 저차원 벡터로 압축할 수 있습니다. PCA와 달리 비선형적인 특징까지 포착할 수 있다는 장점이 있습니다.

---

### 🧩 인코더(Encoder)와 디코더(Decoder)의 개념 및 차이점

오토인코더는 구조적으로 크게 두 부분으로 나뉘며, 입력과 출력이 최대한 유사해지도록 병목(Bottleneck) 구조를 거치게 됩니다.

| 구분 | 인코더 (Encoder) | 디코더 (Decoder) |
| :--- | :--- | :--- |
| **개념** | 고차원의 입력 데이터를 핵심 특징만 남긴 **저차원의 잠재 벡터(Latent Vector/Code)로 압축**하는 단계 | 압축된 잠재 벡터를 받아 **원래의 고차원 입력 데이터 형태로 다시 복원**하는 단계 |
| **데이터 흐름** | 입력 데이터 ($x$) $\rightarrow$ 잠재 공간 ($z$) | 잠재 공간 ($z$) $\rightarrow$ 복원 데이터 ($\hat{x}$) |
| **차원의 변화** | 차원이 점진적으로 **축소됨** (High $\rightarrow$ Low) | 차원이 점진적으로 **확장됨** (Low $\rightarrow$ High) |
| **핵심 목적** | 데이터의 불필요한 노이즈를 제거하고 **가장 중요한 특징(Feature)만 추출**하는 것 | 인코더가 압축한 정보를 바탕으로 **원본과 똑같은 형태로 재구성(Reconstruction)**하는 것 |

* **결론적인 차이점:** 인코더는 정보의 '압축 및 요약'을 담당하고, 디코더는 압축된 요약본을 바탕으로 '원본 재현'을 담당합니다. 손실 함수는 입력값 $x$와 출력값 $\hat{x}$ 사이의 차이인 복원 오차를 최소화하는 방향으로 두 네트워크를 동시에 학습시킵니다.
