# 위클리 페이퍼 #9 — 이미지를 픽셀 단위로 읽다: Image Segmentation

> **이번 주 질문**
> 1. Semantic Segmentation이란 무엇이며, 이미지 분류(Classification)와 어떤 차이가 있나요?
> 2. Fully Convolutional Networks(FCN)의 주요 특징과 기존 CNN 기반 분류 모델과의 차이점은?
>
> **한 줄 답** — 분류가 *"이미지 → 단어 1개"* 라면, 분할은 *"픽셀마다 단어"* 입니다. 그리고 **FCN**은 분류기의 마지막 FC 층을 1×1 합성곱으로 바꿔, 분류기를 *픽셀 예측기*로 탈바꿈시키며 현대 분할의 문을 열었습니다. 이 페이퍼는 분할의 전 과정 — *무엇을·왜·어떻게* — 과 함께, 특히 **헷갈리기 쉬운 핵심 원리(전치 합성곱·RoI Align·IoU vs Dice)** 를 깊이 풀어냅니다.

---

## PART 1. 분할의 지형도 — 분류·검출, 그리고 분할

### 1.1 답하는 정보의 양에 따라 — 분류 → 검출 → 분할

객체 인식은 *"얼마나 자세히 답하느냐"* 에 따라 단계적으로 나뉩니다.

| 작업 | 질문 | 출력 |
| --- | --- | --- |
| **이미지 분류 (Classification)** | 이 사진은 무엇인가 | 라벨 하나 ("고양이") |
| **객체 검출 (Object Detection)** | 무엇이, 어디에 있는가 | 물체마다 박스 + 라벨 |
| **분할 (Segmentation)** | 어느 *픽셀*이 무엇인가 | 물체의 **픽셀 단위 윤곽** |

> [!NOTE]
> **왜 박스로는 부족한가?** 박스는 직사각형이라, 비스듬히 누운 고양이나 구불구불한 도로처럼 *네모나지 않은* 물체에서는 박스 안에 배경이 잔뜩 들어옵니다. *"고양이에 해당하는 부분이 정확히 어디까지인가"* 는 박스로 답할 수 없습니다. 분할은 물체에 해당하는 **픽셀만** 정확히 짚어냅니다.

### 1.2 세 종류의 Segmentation

"픽셀을 *어디까지* 구분하느냐" 에 따라 세 갈래로 나뉩니다.

* **Semantic Segmentation** — 픽셀마다 *클래스*를 칠합니다. 같은 클래스면 전부 같은 색 → 사람이 셋 서 있어도 한 덩어리 '사람 영역'(개체 구분 X). *장면 전체의 구성*이 중요한 곳(위성 영상의 농지·도시·숲 비율, 종양 영역 전체)에서 씁니다.
* **Instance Segmentation** — 같은 클래스라도 *개체*를 따로 떼어냅니다. 사람1·사람2·사람3. *개수 세기·추적·개체별 분석*(매장 손님 수, 세포 개수)에 씁니다.
* **Panoptic Segmentation** — 위 둘을 합칩니다. 핵심은 세상을 **두 부류**로 나눠 다르게 처리하는 것:

> [!IMPORTANT]
> **Thing vs Stuff (파놉틱의 핵심)**
> - **Thing (셀 수 있는 것)** — 자동차·사람처럼 개체로 셀 수 있는 물체 → **Instance 방식**으로 하나하나 구분
> - **Stuff (셀 수 없는 것)** — 도로·하늘·건물처럼 영역으로만 의미 있는 배경 → **Semantic 방식**으로 한 덩어리
>
> 자율주행을 떠올리면 자연스럽습니다 — 앞차·옆차는 따로 추적해야 하니 Thing, 도로·하늘은 영역만 알면 되니 Stuff. 파놉틱은 둘을 한 장면에서 *동시에* 처리합니다.

### 1.3 어디에 쓰이나

자율주행(도로·차선·보행자 실시간), 의료영상(장기·종양·세포), 위성·원격탐사(지형·농지), 산업 검사(웨이퍼·표면 결함), 리테일·문서 스캔, 로보틱스. 픽셀 단위로 *카테고리·위치·모양*을 모두 알려주기에, 장면을 **빠짐없이 이해**해야 하는 분야의 기반이 됩니다.

> [!NOTE]
> **PART 1 한 줄 요약** — 분류=라벨 1개, 검출=박스, **분할=픽셀별 윤곽**. 분할은 *Semantic(클래스)*, *Instance(개체)*, *Panoptic(Thing+Stuff)* 셋으로 나뉜다.

## PART 2. 분류기를 '분할기'로 — FCN

### 2.1 분류 CNN을 그대로 쓸 수 없는 이유

분류 CNN은 **합성곱·풀링으로 특징맵을 점점 줄이며**(224→112→56…) "무엇이 있는지"를 압축하고, 마지막 **FC(Fully Connected) 층**에서 라벨 하나를 냅니다. 이 구조의 본질은 *공간 정보를 버리고 "무엇인지"만 남기는* 것 — 분류엔 유리하지만 분할엔 정반대입니다. 분할은 두 가지 한계에 부딪힙니다.

1. **FC 층의 존재** — 모든 픽셀을 한 줄로 펴 섞으니 **위치 정보가 사라집니다.**
2. **크기가 줄기만 하는 구조** — 작아진 특징맵을 **원래 크기로 되돌릴 방법이 없습니다.**

이 둘을 해결한 모델이 **FCN(Fully Convolutional Network, 2015)** 입니다. 이름 그대로 *"온통 합성곱으로만"* 만든 모델이죠.

### 2.2 합성곱화(Convolutionalization) — FC ≡ Conv

FCN의 첫 아이디어는 마지막 **FC 층을 합성곱으로 대체**하는 것입니다. 놀랍게도 **FC와 합성곱은 같은 계산**입니다.

> [!IMPORTANT]
> **수치로 보는 동치** — 특징맵이 `[512, 7, 7]` 일 때:
> - **FC:** 7×7×512 = **25,088개** 를 한 줄로 펴 가중치 25,088개와 곱해 더함 → 점수 = w₁x₁ + … + w₂₅₀₈₈x₂₅₀₈₈ + b
> - **Conv:** `Conv2d(512, 1, kernel_size=7)` 로 **필터를 특징맵 크기(7×7)와 같게** 잡으면, 필터 가중치도 7×7×512 = **25,088개**. 필터가 전체를 *한 번 덮고* 끝남 → 같은 식.
>
> 가중치 개수도, 곱하고 더하는 방식도 완전히 같습니다. 차이는 단 하나 — **결과를 한 줄로 펴느냐(위치 소실) vs 공간 모양을 유지하느냐(위치 보존)**. FC를 동등한 conv로 바꾸면 위치가 끝까지 살아남고, 덤으로 **입력 크기 제약도 사라집니다**(큰 이미지를 넣으면 출력도 그만큼 커진 *위치별 클래스 점수 지도*). 분류기가 자연스럽게 픽셀 예측기로 변신하는 순간입니다.

### 2.3 인코더-디코더 — 줄였다가 다시 키운다

크기가 줄기만 하는 문제는 **인코더-디코더** 구조로 풉니다.

* **인코더(Encoder)** — 이미지를 줄이며 특징을 뽑습니다(분류 CNN 앞부분과 동일). "무엇이 있는지"를 압축.
* **디코더(Decoder)** — 줄어든 특징맵을 다시 원래 크기로 **키웁니다(업샘플링)**. 들어올 때 크기로 답이 나오니, 픽셀별 클래스 지도를 얻습니다.

오토인코더가 이미지를 *같은 이미지*로 복원했다면, 여기서는 복원 목적지가 **Segmentation Map** 이라는 점만 다릅니다.

### 2.4 ⚙️ 헷갈리는 개념 ① — 전치 합성곱(Transposed Convolution)

업샘플링의 핵심 도구가 **전치 합성곱**(흔히 '디컨볼루션')입니다. 이름이 어렵지만 동작은 *일반 합성곱의 거꾸로* 입니다.

* **일반 합성곱:** 여러 입력 칸 → 필터로 덮어 **하나의 출력값**(다대일, 크기 ↓)
* **전치 합성곱:** **하나의 입력 칸** → 필터를 곱해 **여러 출력 칸에 흩뿌림**(일대다, 크기 ↑). 흩뿌린 영역이 **겹치면 더합니다.**

> [!NOTE]
> **직관** — 입력 칸 하나하나가 *"커널 모양의 도장"* 을 출력 위에 찍는다고 보면 됩니다. 입력값이 크면 진하게, 작으면 옅게 찍히고, 도장이 겹치는 곳은 값이 합쳐집니다. 보폭(stride)을 키우면 도장 간격이 벌어져 출력이 더 커집니다. 단, 겹침이 고르지 않으면 **체커보드(격자) 무늬** 가 생길 수 있어, 실무에서는 *"양선형 업샘플 + 일반 합성곱"* 으로 대체하기도 합니다(DeepLab 계열).

### 2.5 ⚙️ 헷갈리는 개념 ② — 스킵 연결과 FCN-32s/16s/8s

인코더 끝의 특징맵은 너무 작습니다(224→7, **32배** 축소). 이 7×7을 한 번에 224×224로 키우면, 한 칸이 원본의 32×32 영역을 통째로 대표해 **모자이크처럼 뭉개집니다**. 이게 **FCN-32s**(32배를 한 번에)예요.

해법이 **스킵 연결(Skip Connection)** 입니다. 인코더의 *중간 단계*(28×28, 14×14)는 아직 경계 디테일이 살아 있으니, 이를 디코더로 **건너뛰어 전달**해 더해줍니다.

| 버전 | 방식 | 경계 |
| --- | --- | --- |
| **FCN-32s** | 7×7을 한 번에 32배 | 가장 거침 |
| **FCN-16s** | 2배 키운 뒤 14×14 단계를 더하고 16배 | 더 또렷 |
| **FCN-8s** | 28×28 단계까지 더한 뒤 8배 | 가장 또렷 |

> [!NOTE]
> **왜 더하면 또렷해지나?** 깊은 단계(7×7)는 *"무엇인지"* 는 잘 알지만 뭉툭하고, 얕은 단계(28×28)는 *"경계가 어디인지"* 를 잘 압니다. **의미 + 경계** 를 합치면 둘 다 잡힙니다.

> [!NOTE]
> **PART 2 한 줄 요약** — FCN의 한 수는 **FC → 1×1 conv**(공간 보존·임의 입력). 줄인 특징맵은 **전치 합성곱**으로 키우고, **스킵 연결**(32s→16s→8s)로 경계를 살린다.

## PART 3. U-Net과 인스턴스·파놉틱

### 3.1 U-Net — FCN을 잇는 더 정교한 인코더-디코더

같은 2015년에 나온 **U-Net**(의료영상용)은 인코더·디코더가 **좌우 대칭(U자)** 입니다. FCN과의 결정적 차이는 **스킵 연결의 쓰임**입니다.

* **모든 단계에서 스킵** — FCN이 일부 단계만 연결했다면, U-Net은 인코더의 **모든 단계**를 같은 높이 디코더로 연결합니다.
* **Add가 아니라 Concatenate** — FCN은 인코더 특징을 디코더에 **더했지만(Add)**, U-Net은 **이어붙입니다(Concatenate)**.
* **깊은 대칭 디코더** — 디코더가 단순히 키우기만 하지 않고, 이어붙인 정보를 매 단계 합성곱으로 다시 다듬습니다.

> [!IMPORTANT]
> **⚙️ 헷갈리는 개념 ③ — Add vs Concatenate**
> - **Add(더하기, FCN):** 두 특징을 한 덩어리로 *섞어버림*. 채널 수 그대로.
> - **Concatenate(이어붙이기, U-Net):** 둘을 *나란히 쌓아둠*(채널 수 합쳐짐). 섞지 않으니, 뒤따르는 합성곱이 **"어느 쪽 정보를 얼마나 쓸지" 스스로 학습**해 골라 씁니다.
>
> 이 한 끗 덕에 U-Net은 경계를 훨씬 또렷하게 살립니다.

U-Net은 **적은 데이터로도 경계를 정교하게** 잡아냅니다. 강한 데이터 증강(회전·변형)과 맞물려, 데이터가 귀한 **의료영상 분할의 사실상 표준**이 됐습니다(최신 nnU-Net까지 이어짐).

### 3.2 인스턴스 분할 — Mask R-CNN과 RoI Align

개체까지 구분하려면 다른 설계가 필요합니다. 대표가 **Mask R-CNN(2017)** — 2-Stage 검출기 **Faster R-CNN에 '마스크' 가지를 하나 더 얹은** 모델입니다. 출력이 셋: **클래스 · 박스 · 마스크.** 박스로 개체를 먼저 나누니 마스크도 개체별로 나옵니다.

> [!IMPORTANT]
> **⚙️ 헷갈리는 개념 ④ — RoI Align (왜 RoI Pooling으로는 안 되나)**
> 박스 좌표는 회귀 예측이라 *소수점*("가로 12.4~33.7")으로 나옵니다. 그런데 특징맵은 `feature[12][8]` 처럼 **정수 인덱스로만** 값을 읽을 수 있어, 12.4행은 존재하지 않습니다.
> - **RoI Pooling(구식):** 12.4를 12로 **반올림**. 검출(박스·클래스)에는 0.4칸 밀려도 무방했지만, **마스크는 픽셀 단위**라 0.4칸 밀리면 *경계가 통째로 어긋나고* 학습이 불안정해집니다.
> - **RoI Align(해결):** 반올림 대신, 12.4 자리의 값을 **양선형 보간(Bilinear Interpolation)** 으로 직접 계산합니다. 주변 정수 칸 4개를 *거리에 따라 가중평균* — 가까운 칸은 크게, 먼 칸은 작게(각 모서리는 *대각선 맞은편 직사각형 넓이*를 가중치로). 박스가 정수 칸으로 끌려가 밀리는 일이 없어져, 마스크 경계가 제자리에 그려집니다. 이름 그대로 *정렬(Align)* 이죠.

### 3.3 박스 없이 — 1-Stage 인스턴스, 그리고 파놉틱

* **박스 없는 인스턴스:** YOLACT·SOLO는 후보 영역을 따로 추리지 않고 **한 번에 마스크를 예측**해 속도를 끌어올립니다 — 검출의 *"2-Stage는 정확하나 느림, 1-Stage는 빠름"* 구도가 분할에서도 반복됩니다.
* **파놉틱:** **Mask2Former** 가 *마스크드 어텐션*으로 시맨틱·인스턴스·파놉틱을 **하나의 구조로 통합**했습니다(Thing은 인스턴스, Stuff는 시맨틱으로). 자율주행 데이터(Cityscapes)에서 "앞에 차 몇 대 / 도로가 화면의 몇 %" 같은 정보를 한 번에 뽑습니다.

### 3.4 ⚙️ 헷갈리는 개념 ⑥ — 팽창 합성곱·ASPP (해상도를 지키며 넓게 보기)

분할의 근본 딜레마가 있습니다. **넓게 보려면**(맥락 파악) 풀링으로 줄여 **수용영역(receptive field)** 을 키워야 하는데, **줄이면 경계가 뭉개집니다**. **DeepLab 계열**은 이를 **팽창(Atrous/Dilated) 합성곱**으로 풉니다.

* **팽창 합성곱:** 커널 사이에 *구멍(간격)* 을 넣습니다. 3×3 커널을 dilation=2로 쓰면, 파라미터는 9개 그대로인데 **5×5 영역을 커버** — 즉 **다운샘플링 없이 수용영역만 키웁니다.** 해상도를 지키면서 넓게 보는 묘수죠.
* **ASPP(Atrous Spatial Pyramid Pooling):** 여러 dilation 비율(예: 6·12·18)의 팽창 합성곱을 **병렬로** 두어, 작은 물체부터 큰 물체까지 **다중 스케일 맥락**을 한 번에 잡습니다.
* **DeepLabv3+:** ASPP(인코더) + 가벼운 디코더로 경계를 복원해, CNN 시대 시맨틱 분할의 강력한 표준이 됐습니다(초기 DeepLab은 경계 후처리에 **CRF**도 사용).

> [!NOTE]
> **참고: HRNet** — "줄였다 키운다" 대신 **고해상도 표현을 끝까지 유지**하며 여러 해상도 가지를 *병렬로 두고 반복 융합*합니다. 경계·자세처럼 위치 정밀도가 중요한 과제에서 강합니다.

> [!NOTE]
> **PART 3 한 줄 요약** — U-Net은 *전 단계 스킵 + Concatenate*로 경계를 극대화(의료 표준). 인스턴스는 **Mask R-CNN(+RoI Align)**, 파놉틱은 **Mask2Former**, 그리고 DeepLab의 **팽창 합성곱·ASPP**는 *해상도를 지키며 넓게 보는* 대표 해법이다.

## PART 4. 평가와 학습 — 분할은 무엇으로 채점하나

### 4.1 IoU — 분할에서는 '보조'가 아니라 '중심'

**IoU(Intersection over Union)** = 예측 영역과 정답 영역의 **교집합 ÷ 합집합**. 완벽히 겹치면 1, 안 겹치면 0.

> [!NOTE]
> **위상의 변화** — 검출(#8)에서 IoU는 TP/FP를 가르는 *보조 도구*였고 최종 점수는 mAP였습니다. 하지만 **분할에서는 IoU가 곧 정확도 그 자체** — 픽셀을 얼마나 정확히 칠했는지가 성능이니까요.

### 4.2 mIoU — 왜 클래스별로 평균 내나

분할의 표준 지표는 **mIoU(mean IoU)** — 클래스마다 IoU를 따로 구한 뒤 **평균**.

> [!IMPORTANT]
> **클래스 불균형 때문** — 분할 데이터는 배경 픽셀이 대부분입니다. 전체를 뭉뚱그리면 픽셀 많은 배경만 잘 맞춰도 점수가 높게 나옵니다.
> 예: 배경 IoU 0.95, 차 IoU 0.40 → 뭉뚱그리면 ~0.9(잘한 것처럼 보임). **클래스별 평균은 (0.95 + 0.40) / 2 = 0.675** 로 "차를 못 잡고 있다"가 드러납니다. 픽셀 적은 클래스도 동일 비중으로 반영되니 더 공정합니다.

### 4.3 ⚙️ 헷갈리는 개념 ⑤ — IoU vs Dice

둘 다 *겹침*을 재지만 식이 다릅니다.

$$\text{IoU} = \frac{|A \cap B|}{|A \cup B|} = \frac{TP}{TP + FP + FN}, \qquad \text{Dice} = \frac{2\,|A \cap B|}{|A| + |B|} = \frac{2\,TP}{2\,TP + FP + FN}$$

* **관계:** $\text{Dice} = \dfrac{2\,\text{IoU}}{1 + \text{IoU}}$ → **항상 Dice ≥ IoU**(0과 1에서만 일치). Dice가 더 *후하게* 줍니다.
* **차이의 직관:** IoU는 오차를 더 *엄하게* 봅니다(분모에 합집합 전체). Dice는 교집합을 두 번 세 *소수·작은 물체에 덜 가혹* → **손실 함수로 인기**(특히 의료). 반면 **평가 표준 지표는 IoU(mIoU)** 가 보통입니다.
* (참고: 인스턴스는 개체별 품질을 보는 **mask AP**, 파놉틱은 **PQ(Panoptic Quality)** 를 함께 씁니다.)

### 4.4 그 밖의 핵심 지표 — Pixel Accuracy · PQ · Boundary IoU

* **픽셀 정확도(Pixel Accuracy):** 맞춘 픽셀 ÷ 전체 픽셀. 직관적이지만 **불균형에 속기 쉬워**(배경만 맞춰도 높음) 보조로만 씁니다.
* **PQ(파놉틱 품질):** 파놉틱의 표준 지표로, **인식(Recognition)과 분할(Segmentation) 품질을 분리**해 곱합니다.

$$\text{PQ} = \underbrace{\frac{\sum_{(p,g)\in TP}\text{IoU}(p,g)}{|TP|}}_{\text{SQ: 매칭된 영역의 평균 IoU}} \times \underbrace{\frac{|TP|}{|TP| + \tfrac12|FP| + \tfrac12|FN|}}_{\text{RQ: F1형 인식 점수}}$$

  즉 *"얼마나 정확히 칠했나(SQ)"* 와 *"얼마나 빠짐없이 찾았나(RQ)"* 를 동시에 봅니다(매칭 기준은 보통 IoU > 0.5).
* **Boundary IoU / Boundary F1:** 영역 전체가 아니라 **경계 띠(band)** 에서만 IoU를 재, 큰 물체에서 경계 오차가 묻히는 mIoU의 한계를 보완합니다. *경계 품질*이 중요한 의료·정밀 분할에서 함께 봅니다.

### 4.5 손실 함수 — 클래스 불균형과의 싸움

분할 손실의 핵심 과제는 **배경 ≫ 전경** 불균형과 **경계 정밀도**입니다.

| 손실 | 핵심 | 강점 / 언제 |
| --- | --- | --- |
| **Cross-Entropy(CE)** | 픽셀별 로그 손실 | 안정·잘 보정됨. 단 **불균형에 취약**(다수 클래스에 지배됨) |
| **Weighted CE / OHEM** | 클래스 가중 · 어려운 픽셀만 학습 | 불균형·어려운 영역 보강(OHEM = 손실 큰 픽셀만 역전파) |
| **Dice Loss** | 1 − Dice계수 | **불균형에 강건**(겹침 기반). 소수·작은 물체. 단 저겹침 구간 *그래디언트 불안정* |
| **Focal Loss** | 쉬운 픽셀 down-weight | **어려운 픽셀·경계에 집중**(γ 하이퍼파라미터) |
| **Tversky / Focal-Tversky** | FP·FN 가중 분리(α,β) | **재현율 편향** 조절(놓치면 안 되는 의료 등) |
| **Lovász-Softmax** | **IoU(Jaccard)를 직접 최적화** | 평가 지표(mIoU)와 손실의 간극을 줄임 — 미세조정 단계에 효과 |
| **Combo (CE + Dice)** | 두 손실 합 | CE 안정성 + Dice 불균형 강건성 → **실무 베스트 프랙티스** |
| **경계(Boundary) 손실** | 경계 거리 페널티 | **지그재그·미세 경계**(의료) |

> [!NOTE]
> **손실 선택의 직관** — *"평가는 IoU인데 왜 CE로 학습하나?"* 라는 간극을 줄이려는 흐름입니다. **Dice/Lovász** 는 겹침·IoU를 직접 겨냥하고, **CE** 는 안정적 그래디언트를 주므로, 실무는 둘을 **합쳐(CE+Dice)** 안정성과 지표 정합을 모두 취합니다.

> [!NOTE]
> **PART 4 한 줄 요약** — 분할은 **IoU/mIoU**(클래스별 평균, 불균형 대응)로 채점한다. **Dice는 IoU보다 후하고** 소수 물체에 강해 *손실*로 자주 쓴다. 실무는 **CE+Dice 복합 손실**이 표준에 가깝다.

## PART 5. 최신 동향 & 실전 베스트 프랙티스

### 5.1 트랜스포머·파운데이션 시대

CNN 계열(FCN·U-Net·DeepLab)을 넘어, 지금은 **트랜스포머·파운데이션 모델**이 주류입니다.

* **SegFormer / Mask2Former** — 트랜스포머 기반. Mask2Former는 *마스크드 어텐션*으로 세 과제를 통합하며 **Mask R-CNN 대비 약 8× 빠른 수렴**, COCO 파놉틱 **PQ 57.8**(Swin-L).
* **SAM(Segment Anything) 1 → 2 → 3** — **프롬프트(점·박스, 그리고 텍스트) 한 번으로 무엇이든 분할**하는 제로샷 파운데이션 모델. SAM 2는 *영상+메모리*, **SAM 3는 텍스트 기반 오픈 보캐뷸러리**(*presence token* 으로 개념 존재 여부부터 판단).

> [!TIP]
> **💡 SAM 3의 현실 (발표 포인트)** — 화려해 보이지만, *텍스트 프롬프트만* 으로는 도메인 성능이 크게 떨어집니다(한 분석에서 의료 Dice **11.9%**, 추론 시 GT 박스에 의존). YOLO11 대비 **약 55× 느리지만 경계 안정성은 약 12× 우수**. 그래서 현업의 결론은 *"SAM 3의 킬러 앱은 추론 배포가 아니라 **데이터 라벨링**"*. 거대 범용 모델의 강점을 **자동 라벨링**에 쓰는 게 핵심입니다.

### 5.2 오토라벨링 — 값싼 박스를 비싼 마스크로

분할 라벨(픽셀)은 제작비가 매우 큽니다(이미지 한 장 수십 분). 반면 박스 라벨은 흔하죠. **이미 있는 박스 라벨을 SAM에 프롬프트로 넣어 마스크로 변환**하면, 값싼 데이터를 정밀 분할 데이터셋으로 손쉽게 업그레이드할 수 있습니다 — 이것이 실무에서 가장 즉시 유효한 활용입니다.

### 5.3 실전 모델 선택 가이드

| 상황 | 추천 | 이유 |
| --- | --- | --- |
| **의료영상** | **nnU-Net** (CNN) | 자동 구성·소량 데이터에 강함. *트랜스포머가 못 따라오는* 영역 |
| **실시간·엣지** | **YOLO-seg / YOLACT++** | 33+ FPS, 엣지 배포. 정해진 클래스 안에서 빠르게 |
| **최고 정확도** | **Mask2Former** | 통합·SOTA. 단 복잡도가 정당화될 때만 |
| **데이터 라벨링** | **SAM 3** | 오픈 보캐뷸러리 오토라벨링 워크플로우 |

### 5.4 베스트 프랙티스

* **타깃 증강(targeted augmentation)** — 소수 클래스가 나타나는 영역 위주로 증강하면, 인위적 샘플로 데이터를 부풀리지 않고도 소수 클래스 인식이 좋아집니다.
* **복합 손실** — 단일 손실보다 **CE+Dice / Focal-Tversky** 같은 조합이 불균형·경계에 강합니다.
* **올바른 평가** — 단일 픽셀 정확도에 속지 말고 **클래스별 mIoU**(+ 경계·소형 객체 별도 점검)를 보세요.
* **파운데이션 모델은 라벨링에** — SAM류는 추론 배포보다 *데이터 제작 가속*에 먼저 투입.

### 5.5 실전 사례

자율주행(도로·보행자·차량 실시간 분할 → 주행 가능 영역), 의료(MONAI·SAM으로 장기·종양·세포), 위성·원격탐사(지형·작황), **산업 검사**(웨이퍼·금속 표면 스크래치 등 결함 — SAM 3가 "scratches on metal bolts" 같은 결함도 텍스트로 검출), 리테일·문서.

> [!NOTE]
> **PART 5 한 줄 요약** — 트랜스포머·파운데이션(SegFormer·Mask2Former·SAM)이 주류지만, **도메인이 모델을 정한다** — 의료=nnU-Net, 실시간=YOLO-seg, 정확도=Mask2Former, 라벨링=SAM 3. 그리고 파운데이션 모델의 *최고의 실전 용도는 오토라벨링*.

## 마무리 — 발표 핵심 셋

1. **분류 vs 분할** — 분류가 "이미지 → 단어 1개"라면, **분할은 "픽셀마다 단어"**(밀집 예측). 위치를 끝까지 보존해야 하며, 채점은 **mIoU**로 한다.
2. **FCN의 한 수** — 분류기의 **FC를 1×1 conv로 바꿔**(공간 보존·임의 입력) 픽셀 예측기로 변신시킨 것. 줄인 특징맵은 **전치 합성곱**으로 키우고 **스킵**으로 경계를 살린다. 이 골격이 **U-Net**(전 단계 스킵+Concatenate)으로 정교화됐다.
3. **헷갈리는 여섯 가지를 정복하면 끝** — *전치 합성곱(거꾸로 합성곱), 스킵 연결(의미+경계), Add vs Concatenate(섞기 vs 쌓기), RoI Align(반올림 대신 보간), IoU vs Dice(Dice가 더 후함), 팽창 합성곱·ASPP(해상도 지키며 넓게)*. 거기에 채점은 **mIoU**, 그리고 2026년의 무게추는 **트랜스포머·파운데이션 + 오토라벨링**으로 옮겨갔다.

> **한 문장 정리** — 분할은 *"모든 픽셀이 무엇인지"* 를 답하는 기술이고, 그 길은 **FCN의 합성곱화**에서 시작해 **U-Net의 정교한 스킵**, **Mask R-CNN의 RoI Align**, 그리고 **SAM의 프롬프트형 파운데이션**으로 이어졌다.

## 부록 — 세그멘테이션 성능을 끌어올리는 튜닝 대상 총정리

실제로 **mIoU·경계 품질**을 끌어올릴 때 손대는 것들 — 데이터부터 배포까지. `코드체`는 프레임워크에서 자주 만나는 인자명입니다.

**① 데이터 · 증강**
- **해상도·크롭 크기**(`crop_size`): 키우면 경계·소형 구조↑(메모리↓). 분할은 보통 *큰 입력*이 유리.
- **타깃 증강**: 소수 클래스가 나타나는 영역 위주 증강 → 데이터 부풀림 없이 소수 클래스↑.
- **기하·광도 변형**: flip·scale·rotate·elastic(의료)·HSV·CutMix/Copy-Paste(마스크째 합성).
- **라벨 품질·경계 정밀**: 픽셀 라벨 노이즈가 곧 mIoU 천장 → 정제·일관성 점검. **오토라벨링(SAM)** 으로 박스→마스크 보강.
- **클래스 균형**: 리샘플링·클래스 가중·희소 클래스 우선.

**② 모델 아키텍처**
- **백본**: ResNet·ConvNeXt·Swin·MiT(SegFormer) — 표현력 ↔ 속도.
- **디코더/업샘플**: 전치 합성곱 ↔ 양선형+conv ↔ 언풀링(SegNet 인덱스) ↔ PixelShuffle.
- **맥락 모듈**: **ASPP·팽창 합성곱**(DeepLab), **피라미드 풀링**(PSPNet), **고해상도 유지**(HRNet).
- **스킵 융합**: Add(FCN) ↔ Concatenate(U-Net), 전 단계 스킵.
- **어텐션·트랜스포머**: SegFormer·Mask2Former(마스크드 어텐션).
- **Deep Supervision**: 중간 디코더 단계에 보조 손실 → 수렴·경계 안정(U-Net++/nnU-Net).

**③ 손실 함수 · 할당**
- **불균형**: Weighted CE·Focal·Dice·Tversky·**OHEM**.
- **지표 정합**: **Lovász-Softmax**(IoU 직접 최적화), **Combo(CE+Dice)** = 실무 표준.
- **경계**: Boundary/Distance 손실.

**④ 학습 하이퍼파라미터**
- 옵티마이저+`lr`(AdamW·SGD), cosine+warmup, weight decay.
- **정규화**: 고해상도라 배치가 작음 → **GroupNorm/SyncBN**(BatchNorm 통계 불안정 보완).
- **메모리**: AMP(혼합정밀)·gradient checkpointing·accumulation → 고해상도 학습 가능.
- **전이학습**: Freeze / Partial / Full + **LLRD**(대형 백본 미세조정 표준), 사전학습 출처(ImageNet·도메인).
- EMA·교차검증(k-fold)·조기종료.

**⑤ 추론 · 후처리**
- **슬라이딩 윈도우 / Overlap-tile**: 큰 이미지를 겹치게 타일로 추론(U-Net 표준), 경계 미러 패딩.
- **TTA · 멀티스케일 추론**: flip·다중 해상도 평균 → mIoU↑(속도↓).
- **테스트 해상도** 정렬(학습/추론 transform 일치), argmax/임계값.
- **후처리**: CRF(경계 정제, 고전 DeepLab)·형태학 연산·소영역 제거.

**⑥ 평가 · 진단**
- **클래스별 mIoU**(어느 클래스가 약한지) · **Boundary IoU**(경계 품질) · Pixel Acc(보조) · 파놉틱 **PQ(SQ×RQ)** · 인스턴스 **mask AP**.
- 오버레이·FP/FN 갤러리·혼동 행렬로 *체계적 실패* 포착.

**⑦ 배포 · 최적화**
- 양자화(INT8/FP16)·프루닝·증류·TensorRT/ONNX → 엣지·실시간(YOLO-seg).

> **진단 → 처방** — 막연히 다 건드리지 말고 **지표가 가리키는 곳부터**: *소수 클래스 IoU 낮음* → ①타깃 증강·클래스 가중·③Dice/Focal. *경계 흐릿(Boundary IoU 낮음)* → ②ASPP/HRNet·디코더 보강·③경계 손실·⑤CRF/TTA. *큰 물체는 좋은데 작은 게 약함* → ①해상도↑·②다중 스케일(ASPP/PSP). *배경만 잘 맞음* → 픽셀 정확도 대신 **mIoU**로 보고 ③불균형 손실.

**전이학습 — Freeze · Partial · Full + LLRD** (검출과 동일 원리): 데이터 적고 도메인 유사 → **Freeze**(헤드만). 중간 → **Partial**(점진적 해동). 데이터 충분·격차 큼 → **Full** + 낮은 LR·warmup·EMA·**LLRD**(입력층일수록 LR↓, 헤드일수록 LR↑ → 저수준 사전지식 보존, 파국적 망각 방지).

**도메인별 포인트**
- **의료영상**: FN 치명 → Recall/Tversky·재현율 편향. 소량 라벨 → U-Net/nnU-Net·강증강·교차검증·GroupNorm. 경계 정밀 → Boundary 손실·Dice.
- **자율주행**: 실시간(YOLO-seg/실시간 SegFormer)·파놉틱(Thing/Stuff)·다양한 조명/날씨 증강.
- **위성·원격탐사**: 초대형 이미지 → 타일링, 다중 스펙트럼 채널, 소형 객체 다수.
- **산업 검사**: 결함 작고 희소 → 고해상도·타일링·불균형 손실·합성 결함 증강. FN 치명.

## 심화 — Q&A로 정리한 고급 개념 (Deep-Dive)

> 강의 범위를 넘어, 직접 파고들며 정리한 질문들. FCN의 디테일부터 모던 아키텍처(Transformer·YOLO-Seg·SAM)의 정보 처리, 구조 용어 위계까지. (본문 PART 1~5의 보충·심화)

### D.1 `Conv2d(512, 1, kernel=7)`의 진짜 의미 — 가운데 1은 "공간 1×1"이 아니다

PyTorch `nn.Conv2d(in_channels, out_channels, kernel_size)` 기준으로 정확히 뜯어보면:
- **512 = in_channels**: 들어오는 특징맵의 채널 깊이.
- **kernel=7 = kernel_size**: 필터의 공간 크기 7×7.
- **1 = out_channels**: 출력 **채널(필터) 개수가 1개**라는 뜻. ← 흔한 오해: "공간을 1×1로 만든다"가 **아님**.

그럼 결과 공간이 왜 1×1이 되나? **입력 7×7 = 커널 7×7**이라 필터가 딱 한 번 덮고 끝나기 때문(7×7 도화지에 7×7 도장 → 이동할 자리 없음). 만약 10-클래스라면 `Conv2d(512, 10, kernel=7)` → 출력 텐서 `[10, 1, 1]`. **가중치 수 7×7×512=25,088개는 FC와 동일** — 다른 건 결과를 한 줄로 펴느냐(FC, 위치 소실) vs 공간 차원을 유지하느냐(Conv, 위치 보존)뿐.

### D.2 "정확한 픽셀 위치"는 어떻게 복원되나 — 채널(What) ⟂ 공간격자(Where)

두 정보의 역할 분담이 핵심:
- **채널(C) = What**: "여기에 고양이가 있나? 바퀴가 있나?" 같은 의미. 깊은 레이어일수록 추상적·풍부.
- **공간격자(H×W) = Where**: "그게 어느 위치인가." 얕은 레이어일수록 정밀.

전치합성곱(Transposed Conv)만으로 7×7을 한 번에 224×224로 키우면(FCN-32s) 대략 위치는 가지만 경계가 뭉툭(coarse)하다. **정밀 픽셀 좌표는 Skip Connection이 복원**한다 — 얕은 레이어의 고해상도 위치 특징을 끌어와 더함(FCN-16s/8s). 즉 "의미를 잘 아는 깊은 정보 + 위치를 잘 아는 얕은 정보"의 **결합**이 핵심(채널이 쪼개져서가 아니라 두 정보의 융합 덕분).

### D.3 Skip Connection이 넘기는 값의 정체 — 원본 픽셀이 아니다

흔한 오해: Skip으로 넘어가는 게 원본 픽셀값(0~255)? **아니다.**
- 넘어가는 건 **특징맵(Feature Map) 텐서 C×H×W 전체**(큐브 통째 복사).
- 칸 안의 숫자는 **활성화 값(Activation)** — 합성곱+활성화(ReLU 등)를 거친 실수. ReLU면 범위 **0~∞**(0~1/0~255에 갇히지 않음. BatchNorm으로 분포는 정돈).
- 의미는 **"이 위치에 이 패턴이 얼마나 강하게 있나"의 반응 점수(Response Score)**. 예: 1번 채널=수직선 검출기, 그 위치 값 15.8 → 거기에 뚜렷한 수직선. 깊은 채널은 '바퀴'·'눈' 같은 의미 특징의 강도.

→ Skip이 넘기는 건 RGB 물감이 아니라 **"위치별 특징 점수표(3D 리포트)"**. 디코더는 이걸 참고해 흐릿한 경계를 제자리에 선명히 복원.

### D.4 인코더 피처맵엔 공간+채널이 한 몸 — 그런데 왜 Skip이 필요한가

맞다, 인코더의 모든 피처맵은 항상 C×H×W로 **공간(Where)과 채널(What)이 분리되지 않고 공존**한다. 문제는 **정보의 질**이 깊이에 따라 극단적으로 변한다는 것:

| 레이어 | 공간(H,W) | 채널(C) |
| --- | --- | --- |
| 얕음 (예 64×112×112) | **정밀**(픽셀 1:1, 테두리·질감) | 단순(선·반점 수준) |
| 깊음 (예 512×7×7) | **뭉개짐**(대략 사분면) | **풍부**(고양이 귀·바퀴 등 의미) |

"완벽 정밀 공간 + 고도 추상 의미"를 **동시에 100% 가진 단일 피처맵은 없다.** 분류 헤드는 깊은 피처맵만으로 충분(GAP로 공간 짓눌러 의미만). 마스크 헤드는 뭉개진 7×7론 털을 못 오림 → **얕은 레이어의 선명한 위치 정보를 Skip으로 빌려온다.** (깊은 = 시력 나쁜 똑똑이, 얕은 = 시력 2.0의 눈 → 그 눈을 빌린다.)

### D.5 멀티-브랜치(헤드)에서 Skip은 어디로 — 그리고 Add vs Concat

헤드가 공간/마스크/채널로 전문화된 구조(YOLACT·CondInst·Mask R-CNN 변형 등)에서:
- **공간·마스크 헤드 = Skip 필수**: 정밀 경계가 생명 → 얕은 고해상도 Skip이 반드시 흘러듦(또는 분기 직전 메인 디코더 스트림에 병합).
- **채널 헤드 = 선택적/생략**: 전역 문맥(GAP로 공간 압축)을 다루니 정밀 좌표 Skip은 연산 낭비 → 보통 생략하거나 간접 활용.

융합 방식 비교:

| 방식 | 구조 | 특징 |
| --- | --- | --- |
| **Add** (FPN·ResNet) | 채널 수 **일치** 필요(다르면 1×1 conv로 맞춤) | 같은 공간서 블렌딩, 채널 유지 → **메모리 효율** |
| **Concat** (U-Net·DenseNet) | 채널축으로 붙임 → 채널 **≈2배** | 두 정보 안 섞이고 보존, 뒤 conv가 선택 학습 → **표현력↑, 메모리↑** |

### D.6 업샘플링 커널 크기 — 2×2 / 3×3 / 4×4 / 1×1, 그리고 resize-conv

인코더가 보통 stride 2로 절반씩 줄이니 복원도 stride 2로 2배씩이 자연스럽다. **체커보드 아티팩트(checkerboard)**의 진짜 원인은 **커널 크기가 스트라이드로 나눠떨어지지 않을 때 생기는 "불균일 중첩(uneven overlap)"**(Odena et al. 2016):

| 커널/stride | 중첩 | 특징 |
| --- | --- | --- |
| **2×2 / s2** | 비중첩(딱 맞음) | 나눠떨어짐 → 불균일중첩 없음. 단 블렌딩이 없어 **블록감**, 무작위 초기화 시 잔여 아티팩트 가능 |
| **3×3 / s2** | 불균일 | 3은 2로 안 나눠떨어짐 → **전형적 체커보드(최악)** |
| **4×4 / s2** | 균등 중첩 | 나눠떨어지고 중앙이 고르게 겹침 → **부드러움, 전치합성곱의 표준 선택** |
| **1×1 / s1** | — | 공간 복원 **불가**(H×W 못 키움). **채널 맞추는 프로젝션(projection) 전용** |

**최신 트렌드 — resize-conv**: 전치합성곱 대신 최근접/쌍선형(Bilinear)으로 2배 키운 뒤 3×3 일반 합성곱으로 정제. 아티팩트를 근본적으로 회피해 4×4 전치합성곱보다도 선호되는 추세.

> ⚙️ 교정 포인트: "2×2면 체커보드"는 부정확. 2×2·4×4 모두 stride로 나눠떨어져 불균일중첩은 없다. 진짜 체커보드는 3×3처럼 **안 나눠떨어질 때**. 차이는 2×2=비중첩 블록감 vs 4×4=중첩으로 더 매끄러움.

### D.7 모던 아키텍처는 공간·채널을 어떻게 다루나 — CNN / Transformer / YOLO-Seg / SAM

CNN은 C×H×W "격자 큐브"를 물리적으로 유지하지만, 모던 아키텍처는 격자를 깨고 토큰·쿼리·프롬프트로 다룬다.

**① Transformer (Swin · Mask2Former)**
- 인코더(Swin): 격자 → **N×C 토큰 행렬**(N=패치 수). 격자가 깨져 위치를 모르니 **위치 인코딩(Positional Encoding)**을 채널 값에 더해 주입. **Self-Attention**으로 이미지 끝-끝 패치까지 한 번에 의미 교환. Swin은 로컬 윈도우+토큰 병합으로 계층화.
- 디코더(Mask2Former): Skip 대신 **Object Queries(N개 벡터)** 사용. 쿼리가 인코더 토큰에 **Cross-Attention**으로 "고양이 특징 가진 너희 위치 줘" 질의 → 채널+공간 흡수해 마스크 생성.

**② YOLO-Seg (YOLOv8/11-Seg)**
- 인코더·버틀넥: CNN(CSPDarknet)+**PANet**으로 위↔아래 Skip을 무수히 교차 → 모든 레이어가 강한 공간+채널.
- 디코더: U-Net식 점진 복원 대신 **공간·채널 완전 분리**(YOLACT식) — **Protonet 브랜치**가 공간 마스크 템플릿 K개, **Coefficient 브랜치**가 조합 계수 K개 → 마지막에 **(템플릿)×(계수) 행렬곱**으로 마스크 단번 생성.

**③ SAM (Segment Anything, 파운데이션)**
- 인코더: 무거운 **ViT**가 고해상도를 한 번에 256×64×64 **Image Embedding**으로 압축. **Skip 없음** — MAE 사전학습으로 그 격자 안에 공간+의미가 이미 융합.
- 버틀넥(Prompt Encoder): 점·박스·텍스트 → C차원 토큰(질의)로 번역.
- 디코더(경량 Transformer): **Two-way Attention**(Prompt↔Image)으로 임베딩에서 필요한 부분만 빼 마스크 렌더.

| 계열 | 공간(Where) | 채널(What) | 핵심 융합 |
| --- | --- | --- | --- |
| 전통 CNN | 격자 H×W + Skip | 깊이 C | Concat / Add |
| Transformer(M2F) | Positional Encoding | 토큰 벡터 | Cross-Attention |
| YOLO-Seg | Protonet 템플릿 | 마스크 계수 | 선형결합(행렬곱) |
| SAM | 프롬프트로 직접 지시 | ViT 임베딩에 내재 | Two-way Attention |

### D.8 구조 용어 위계 — 백본·넥·헤드·브랜치·태스크아키텍처·모델 (자동차 공장 비유)

논문·코드에서 마구 섞이는 용어를 위계로 정리:
1. **백본(Backbone)** = 엔진·차대: 특징 추출기(ResNet·ConvNeXt·ViT·Swin). 보통 ImageNet 사전학습.
2. **넥(Neck)** = 변속기: 다중 스케일 특징 융합(FPN·PANet·BiFPN). Skip이 가장 복잡하게 얽히는 구간.
3. **헤드(Head)** = 바퀴·조향: 최종 출력단(Class/Bbox/Mask Head). Loss 직전.
4. **브랜치(Branch)** = 병렬 회로: 헤드 안/중간에서 역할별로 갈라진 경로(Protonet·Coefficient 브랜치). 하나의 헤드에 여러 브랜치 가능.
5. **태스크 아키텍처(Task Architecture)** = 차종(SUV/세단): 거시 패러다임 — Two-Stage(Mask R-CNN) / Single-Stage(YOLACT·YOLO-Seg) / Query-based(Mask2Former).
6. **모델(Model)** = 완성차명: 조립+학습 완료 결과물(`yolov8x-seg.pt`).

**실전 분해**
- **Mask R-CNN**: Two-Stage 인스턴스 · 백본 ResNet-50 · 넥 FPN · 1단계 RPN · 2단계 RoI Align 후 Class/Bbox/Mask **브랜치**.
- **YOLOv8-Seg**: Single-Stage(anchor-free) · 백본 CSPDarknet · 넥 PANet · Decoupled Head(Protonet+Coefficient 브랜치 → 행렬곱).
- **Mask2Former**: Universal(Query-based) · 백본 Swin · 넥 Pixel Decoder · 헤드 Transformer Decoder(Object Queries + Cross-Attention, Skip 대신 쿼리).

**혼동 방지 치트시트**

| 코드/논문에서… | 용어 | 예시 |
| --- | --- | --- |
| 사전학습 특징 추출기를 가져옴 | 백본 | ResNet·Swin-T |
| 여러 스케일 특징맵을 섞는 층 | 넥 | FPN·PANet |
| Loss 직전 최종 출력단 | 헤드 | Mask Head |
| 입력을 쪼개 병렬 연산 | 브랜치 | Protonet Branch |
| 박스 먼저냐 한번에냐 설계 | 태스크 아키텍처 | Two/Single/Query |
| 다 조립해 학습한 .pt | 모델 | yolov8x-seg.pt |
