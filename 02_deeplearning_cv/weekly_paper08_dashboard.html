검출을 '회귀 문제'로 재정의하다

🧹 NMS 시연 — 중복 박스는 어떻게 정리될까

YOLO는 한 객체에 중복 박스를 여러 개 만듭니다. 추론 마지막에 NMS(Non-Maximum Suppression)로 신뢰도 1위만 남기고 나머지를 제거하죠. 이 후처리가 지연(Latency)의 주범이었습니다.

최신 동향

한때 NMS-free는 트랜스포머 계열(RT-DETR · D-FINE · DEIM)의 전유물이었습니다. 이제는 YOLO 본가가 흡수했습니다 — YOLOv10이 길을 열고, YOLO26이 네이티브 NMS-free를 기본 탑재하면서 "NMS 없는 실시간 검출"이 표준 디폴트가 됐습니다.

Part 1 · 인터랙티브 시뮬레이터

가장 헷갈리는 지점 — Confidence와 두 종류의 IoU(‘중복 제거용’과 ‘채점용’)가 각각 어디서, 무슨 일을 하는지를 슬라이더로 분리해서 봅니다. NMS 적용 ↔ NMS-free를 토글하면 어떤 노브가 살아있고 어떤 노브가 죽는지가 그대로 드러납니다.

핵심

“IoU가 사라진다”가 아닙니다. NMS-free 모델(YOLO26 등)은 학습 단계의 일대일 매칭으로 객체당 박스를 처음부터 1개만 내보내므로, 추론 파이프라인에서 중복 제거용 NMS IoU 노브를 돌릴 일이 없을 뿐입니다. 성능을 채점하는 평가 IoU는 두 모드 모두에서 똑같이 살아있습니다 — 사라지는 건 ‘IoU’가 아니라 ‘추론 시 튜닝하던 NMS IoU 임계값’입니다.

실험 가이드

① NMS IoU를 0.7↑로 올려보세요 — 중복 박스가 되살아나 FP가 늘어납니다. ② 모드를 NMS-free로 바꾸면 ② 슬라이더가 비활성화되고 중복이 애초에 없습니다. ③ 평가 IoU를 0.65↑로 올리면, 위치가 어설픈 박스가 같은 자리인데도 TP→FP로 뒤집힙니다(두 모드 공통). ④ Confidence를 0.6↑로 올리면 배경 오탐이 사라집니다.

Part 1 · 강점과 한계

왜 빠르고, 무엇이 어려운가

Speed

실시간 추론

단일 신경망이라 연산 낭비가 적어 FPS가 높습니다. 엣지·온디바이스에서 끊김 없는 스트리밍 처리.

Context

글로벌 컨텍스트

이미지 전체를 한 번에 보므로 배경-객체 관계까지 학습 → 빈 배경 오탐(FP)이 적습니다.

Robust

강한 일반화

예술 작품 등 학습 분포 밖(OOD) 데이터에서도 객체의 본질적 특징을 잘 잡아냅니다.

전통적 한계와 극복

Limitation 01

소형 객체 검출

다운샘플링으로 해상도가 손실돼 작은 물체(새 떼·먼 차량)가 뭉개집니다. → FPN 고도화 · 다중 스케일, 그리고 YOLO26의 STAL(소형 객체 인지 라벨 할당)로 학습 단계부터 보강.

Limitation 02

NMS 연산 병목

중복 박스 제거용 NMS는 CPU 바운드로 동작해 지연을 키웠습니다. → 위에서 본 대로 NMS-free 설계로 정면 돌파.

Part 2 · 1단계 — 합격 판정

IoU 실험실

IoU(Intersection over Union)는 정답(GT) 박스와 예측 박스가 얼마나 겹치는지를 잰 값입니다. 예측 박스(시안)를 드래그하고 크기를 조절하면서, IoU가 어떻게 변하고 임계값에 따라 TP / FP 판정이 뒤집히는지 직접 보세요.

0.00
IoU = 교집합 ÷ 합집합
판정: FP
판정조건의미
TP신뢰도 ≥ 임계값 이고 IoU ≥ 기준값실제 객체를 올바르게 찾아냄
FPIoU < 기준값(배경 오인) 또는 중복 박스오탐 — 헛것을 짚음
FN정답인데 못 찾음 / 신뢰도 미달미검출 — 있는 객체를 놓침
연결

같은 객체에 박스가 여럿이면 IoU 최고 하나만 TP, 나머지는 FP입니다. §02의 NMS가 바로 이 중복 FP를 줄이는 장치라는 점이 여기서 이어집니다.

시소

임계값을 낮추면 Recall↑ 이지만 FP가 늘어 Precision↓. 높이면 Precision↑ 이지만 놓쳐서 Recall↓. AP는 이 곡선 아래 면적으로, 임계값과 무관한 종합 점수입니다.

이 그래프가 곧 ‘AP 그래프’입니다

맞습니다 — 위 PR 곡선과 AP는 별개의 그래프가 아닙니다. AP = PR 곡선 아래 면적이고, 콘솔의 음영이 바로 그 면적입니다. §06의 AP·mAP는 이 면적을 IoU 임계값별·클래스별로 반복해 평균 낸 것일 뿐이에요.

바로잡기

SSD는 2-stage가 아니라 YOLO와 같은 1-stage(Single Shot Detector)입니다. 한 번의 통과로 예측하면 1-stage(YOLO·SSD·RetinaNet), 영역 제안을 먼저 하면 2-stage(Faster R-CNN 계열)예요. 그래서 PR 곡선을 실무에 적용할 때 진짜 갈리는 축은 ‘YOLO vs SSD’가 아니라 ‘1-stage vs 2-stage’, 그리고 같은 1-stage 안에서의 ‘YOLO vs SSD 설계 차이’입니다.

PR 곡선을 실무에 적용할 때

1-stage · YOLO / SSD

배경이 압도적으로 많다

그리드·앵커마다 예측이 쏟아져 배경(음성) 후보가 양성보다 수백 배 많습니다. 낮은 conf 구간에서 FP가 빠르게 늘어 PR 곡선이 잘 무너집니다. → conf·NMS IoU 튜닝과 Focal Loss류 불균형 대응이 핵심.

2-stage · Faster R-CNN

후보를 먼저 걸러둔다

RPN이 배경을 1차로 제거해 후보 수가 적습니다. 같은 Recall에서 Precision이 더 높게 유지되고 특히 고IoU(AP₇₅↑) 영역에 강함. 대신 느려서 실시간엔 불리.

YOLO ↔ SSD

둘 다 1-stage지만 설계가 다릅니다. SSD는 여러 해상도의 피처맵 + default box(앵커)에서 예측해, 얕은 피처의 해상도 한계로 소형 객체에 약하고 default box의 스케일·종횡비가 핵심 하이퍼파라미터입니다. YOLO는 그리드 기반에서 anchor-free·FPN/PAN·NMS-free로 진화해 속도/정확도 균형이 더 좋아졌고요. 소형 객체 AP(APS)가 낮을 때 — SSD라면 입력 해상도↑·얕은 피처맵/FPN 보강·default box 재설계, YOLO라면 imgsz↑·P2 헤드 추가·증강 조절이 1순위 처방입니다.

Part 2 · 3단계 — 최종 성적표

AP에서 mAP로, 그리고 IoU 엄격도

AP는 한 클래스(예: '자동차')의 PR 곡선 아래 면적입니다. 모든 클래스의 AP를 평균 내면 최종 학점, mAP가 됩니다.

mAP = 1N Σ APi   (N = 클래스 수)

단일 IoU 기준 비교 — AP₅₀ · AP₇₅ · AP₉₀, 그리고 평균

같은 모델이라도 IoU 합격선을 어디에 두느냐에 따라 점수가 크게 달라집니다. 막대 높이는 각 IoU에서의 대표 AP값입니다. 버튼으로 기준을 바꿔보세요.

선택 기준0.00

AP₅₀ · PASCAL VOC

IoU 0.5. 박스가 헐거워도 클래스만 맞으면 OK. 빠른 점검·프로토타이핑용.

AP₇₅ · COCO 보조

IoU 0.75. 꽤 엄격. COCO가 AP₅₀과 함께 보고하는 정밀 지표.

AP₅₀:₉₅ · COCO 주력

0.50~0.95를 0.05 단위 10번 평가해 평균. 단일 기준의 운에 좌우되지 않음.

AP₅₀₋₉₅ vs mAP — 평균을 내는 '축'이 다르다

둘 다 "평균"이라 헷갈리지만, 어느 방향으로 평균을 내느냐가 다릅니다. 아래는 클래스(세로) × IoU 임계값(가로) 표의 AP값입니다. 버튼을 눌러 어디를 평균 내는지 보세요.

정리

AP₅₀₋₉₅은 한 클래스를 IoU 축으로 평균(가로줄), mAP는 한 IoU에서 클래스 축으로 평균(세로줄)입니다. COCO의 대표 지표(보통 그냥 "mAP" 또는 "AP")는 두 축을 모두 평균한 표 전체 평균이라, 둘이 한 숫자로 합쳐져 더 헷갈립니다.

진단

크기별 APS / APM / APL 를 함께 보세요. APS 만 유독 낮다면 소형 객체 문제 → §03의 FPN 고도화·고해상도 입력이 정답에 가깝습니다.

Wrap-up · 발표 핵심

가져갈 메시지 셋

01

YOLO = 한 번에 회귀

한 번의 추론으로 위치+클래스를 푼다. 그래서 빠르고, 글로벌 컨텍스트를 보고, 잘 일반화한다.

02

mAP = 쌓아 올린 성적표

IoU·신뢰도 → P/R → AP → mAP. 현장 목표(속도냐 정밀도냐, FP냐 FN이냐)에 따라 지표와 임계값을 다르게 잡는다.

03

2026 = NMS-free 표준화

트랜스포머의 전유물이던 NMS-free를 YOLOv10·YOLO26이 본가로 흡수. 실시간 검출의 기본기가 한 단계 올라섰다.

한눈 정리 — YOLO가 "무엇을·어디에"를 한 번에 답한다면, mAP는 그 답이 "얼마나 정확한가"를 채점합니다. 평가 체인은 IoU·신뢰도 → Precision·Recall → PR곡선 → AP → mAP 로 이어지며, 실무에선 AP₅₀로 방향을 잡고 AP₅₀₋₉₅로 정밀도를 끌어올린 뒤, 현장 리스크(FP↔FN)에 맞춰 임계값을 조율합니다.
