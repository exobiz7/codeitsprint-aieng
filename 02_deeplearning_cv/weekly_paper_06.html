Question 01

CNN을 구성하는 각 레이어의 역할에 대해 설명해보세요

한 줄 요약

CNN은 Convolution → Activation → Pooling을 반복해 이미지의 특징을 점점 추상화하고, 마지막에 Flatten → Fully Connected로 최종 예측을 만드는 구조입니다.

왜 CNN인가? — 일반 신경망의 한계

28×28짜리 작은 이미지도 픽셀을 일렬로 펼치면 784개의 입력이 됩니다. 여기에 Fully Connected 레이어를 바로 적용하면 파라미터 수가 폭발적으로 늘어나고, 이미지의 공간 정보(위치 관계, 지역 패턴)를 완전히 잃어버립니다. CNN은 이 문제를 지역 수용 영역(local receptive field)과 파라미터 공유(parameter sharing)로 해결합니다.

레이어별 흐름

아래 파이프라인은 실제 CNN 순전파(Forward Pass)의 전형적인 흐름입니다. 각 레이어가 무엇을 입력받고 무엇을 출력하는지 함께 읽어보세요.

Conv
합성곱 레이어 (Convolutional Layer)
핵심 레이어입니다. 작은 필터(커널)를 이미지 위로 슬라이딩하며 내적(element-wise 곱의 합)을 계산해 특징 맵(Feature Map)을 생성합니다. 하나의 필터는 "이 위치에 특정 패턴이 있는가?"를 감지하는 탐지기 역할을 합니다. 여러 필터를 사용하면 여러 종류의 패턴을 동시에 탐지할 수 있어요. 파라미터는 필터 가중치만이므로 Fully Connected 대비 극적으로 적습니다.
Act
활성화 함수 (Activation Function) — 주로 ReLU
Convolution의 결과(선형 변환)에 비선형성을 부여합니다. ReLU(max(0, x))가 사실상 표준입니다. 음수는 0으로 만들고 양수는 그대로 통과시켜, 네트워크가 복잡한 비선형 패턴을 학습할 수 있게 합니다. Sigmoid보다 기울기 소실(Vanishing Gradient) 문제가 훨씬 적어 깊은 망에 적합합니다.
Pool
풀링 레이어 (Pooling Layer) — 주로 Max Pooling
특징 맵의 크기를 줄여 계산량을 낮추고 주요 특징만 남기는 다운샘플링 레이어입니다. Max Pooling은 지정된 영역에서 최댓값만 취합니다. 이 과정에서 특징의 위치가 약간 변해도 탐지가 유지되는 위치 불변성(Translation Invariance)을 얻습니다. "고양이 귀가 사진 왼쪽에 있든 오른쪽에 있든 귀로 인식"하게 해주는 원리가 이것입니다.
Norm
배치 정규화 (Batch Normalization) — 선택적
레이어 출력을 미니배치 단위로 정규화해 학습을 안정화하고 더 높은 학습률을 사용할 수 있게 합니다. 내부 공변량 이동(Internal Covariate Shift)을 줄여 깊은 망에서의 수렴 속도를 높이고, 과적합을 억제하는 정규화 효과도 있습니다.
Flat
플래튼 (Flatten)
3차원 특징 맵(높이 × 너비 × 채널)을 1차원 벡터로 펼칩니다. Convolutional 블록과 Fully Connected 레이어를 연결하는 다리 역할입니다. 연산 자체는 없고 형태(shape)만 변환합니다.
Drop
드롭아웃 (Dropout) — 선택적, 주로 FC 앞뒤
학습 중 무작위로 일정 비율의 뉴런을 비활성화해 과적합을 방지합니다. 추론(Inference) 시에는 모든 뉴런을 사용합니다. 모든 뉴런이 독립적으로 특징을 학습하도록 강제하는 일종의 앙상블 효과를 냅니다.
FC
완전 연결 레이어 (Fully Connected Layer)
Flatten된 벡터를 입력받아 최종 예측을 수행합니다. Conv 블록이 "특징을 추출"한다면, FC 레이어는 그 특징들을 종합해 "이 특징 조합이 어떤 클래스인가"를 판단합니다. 마지막 FC 레이어의 출력 수는 클래스 수와 같고, Softmax를 통해 확률로 변환합니다.

깊어질수록 추상화된다 — 계층적 특징 학습

CNN의 핵심 원리는 계층적 특징 학습입니다. 얕은 레이어(초기)는 선·모서리·색상 같은 저수준 특징을 감지하고, 깊어질수록 눈·코·귀 같은 부품, 더 깊어지면 "얼굴" "자동차" 같은 고수준 개념을 인식합니다.

레이어 주요 역할 핵심 파라미터
Convolution 필터로 지역 특징 탐지 → 특징 맵 생성 필터 크기, 필터 수, Stride, Padding
Activation (ReLU) 비선형성 부여, 기울기 소실 완화 —
Pooling 다운샘플링, 위치 불변성, 계산량 감소 풀 크기, Stride
Batch Norm 학습 안정화, 수렴 속도 향상 γ, β (학습 가능)
Flatten 3D → 1D 형태 변환 (연산 없음) —
Dropout 과적합 방지, 앙상블 효과 드롭 비율 p
Fully Connected 특징 종합 → 최종 클래스 예측 출력 뉴런 수, 활성화 함수
면접 포인트

CNN이 일반 Fully Connected보다 이미지에 적합한 이유는?

→ 두 가지로 답하세요. ① 파라미터 공유(필터 가중치를 전체 이미지에 공유)로 파라미터 수를 극적으로 줄이고, ② 지역 연결성(local connectivity)으로 공간 정보를 보존합니다. FC는 모든 픽셀이 모든 뉴런에 연결되어 공간 구조를 무시합니다.

Max Pooling과 Average Pooling의 차이는?

→ Max Pooling은 해당 영역에서 가장 강한 특징(최댓값)을 취하고, Average Pooling은 평균을 취합니다. 일반적으로 Max Pooling이 특징의 존재 여부를 감지하는 데 더 유리해 더 자주 쓰입니다.

Stride와 Padding은 무엇인가요?

→ Stride는 필터가 한 번에 이동하는 칸 수입니다. Stride가 크면 출력 특징 맵이 작아집니다. Padding은 입력 가장자리에 0을 추가해 특징 맵의 크기를 유지('same')하거나 줄어들지 않게 하는 기법입니다. 가장자리 픽셀이 무시되지 않게 보호하는 역할도 합니다.

Question 02

오토인코더가 적합한 상황과 인코더·디코더의 개념 및 차이점

한 줄 요약

오토인코더는 입력을 압축(인코더)했다가 복원(디코더)하는 비지도 학습 모델입니다. 레이블 없이 데이터의 핵심 표현을 학습하며, 이상 탐지·차원 축소·노이즈 제거에 특히 강합니다.

오토인코더란?

오토인코더(Autoencoder)는 입력을 그대로 출력으로 재구성하는 신경망입니다. 단순히 복사하는 게 아니라, 중간에 병목(Bottleneck)을 두어 입력을 저차원 잠재 공간(Latent Space)으로 압축했다가 다시 복원하는 과정에서 데이터의 핵심 구조를 학습합니다.

Input X → Encoder → z (Latent Code) → Decoder → X̂ (Reconstruction) 손실 함수: L = ‖X − X̂‖² (입력과 복원 결과의 차이를 최소화)

인코더(Encoder) — 압축과 추상화

인코더는 고차원 입력 데이터를 저차원 잠재 벡터 z로 매핑하는 함수입니다. 입력의 모든 정보를 그대로 담을 수 없는 병목 구조 덕분에 모델이 "덜 중요한 세부 사항을 버리고 본질적인 특징만 보존"하도록 강제됩니다. 이미지라면 픽셀 하나하나가 아니라 "이 이미지가 어떤 모양인지"의 추상적 표현이 z에 담기는 셈입니다.

디코더(Decoder) — 복원과 생성

디코더는 잠재 벡터 z를 입력받아 원래 차원의 데이터로 복원하는 함수입니다. 인코더와 대칭적인 구조를 가지며, 보통 역합성곱(Transposed Convolution)이나 업샘플링을 사용합니다. 학습이 잘 된 디코더는 잠재 공간에서 새로운 z를 입력받아 새로운 데이터를 생성하는 것도 가능합니다 (VAE의 원리).

인코더 vs 디코더 — 핵심 차이

Encoder
압축 · 분석
  • 고차원 → 저차원
  • 정보를 압축·추상화
  • Conv / Dense (수축)
  • 데이터의 핵심 표현 추출
  • 분류·검색에 활용 가능
Decoder
복원 · 생성
  • 저차원 → 고차원
  • 정보를 복원·재구성
  • Transposed Conv (확장)
  • 잠재 표현으로 데이터 생성
  • 생성·복원에 활용 가능

오토인코더가 적합한 상황

오토인코더는 레이블이 없는 데이터에서 특히 빛납니다. 정답 없이 입력 자체를 목표로 학습하므로, 라벨링 비용이 높거나 불가능한 상황에서 강력한 도구가 됩니다.

Use Case 01
이상 탐지 (Anomaly Detection)
정상 데이터만으로 학습 후, 복원 오차가 큰 샘플을 이상치로 판단합니다. 제조 결함 탐지, 금융 사기 감지, 네트워크 침입 탐지 등에 활용됩니다. 정상 데이터 외에 이상 데이터 레이블이 필요 없다는 점이 핵심입니다.
Use Case 02
차원 축소 (Dimensionality Reduction)
PCA보다 비선형 구조를 더 잘 포착합니다. 잠재 벡터 z를 시각화하거나, 후속 분류 모델의 입력으로 사용할 수 있습니다. 고차원 데이터(이미지·텍스트)의 임베딩 추출에 유용합니다.
Use Case 03
노이즈 제거 (Denoising)
노이즈가 섞인 입력을 받아 깨끗한 원본을 복원하도록 학습(Denoising Autoencoder)합니다. 이미지 복원, 오디오 잡음 제거, 결측 데이터 복구 등에 적용됩니다.
Use Case 04
생성 모델 기반 (VAE)
Variational Autoencoder(VAE)로 확장하면 잠재 공간을 확률 분포로 모델링해 새로운 데이터 생성이 가능합니다. 이미지 생성, 데이터 증강, GAN 학습 초기화 등에 활용됩니다.

적합하지 않은 상황도 알아두세요

레이블이 충분히 존재한다면 지도 학습이 훨씬 효과적입니다. 오토인코더의 잠재 표현이 항상 사람이 해석할 수 있는 의미 있는 구조를 보장하지는 않습니다. 이미지 분류처럼 목표가 명확한 경우엔 CNN + Softmax 직접 학습이 더 낫습니다.

면접 포인트

오토인코더가 지도 학습이 아닌 이유는?

→ 학습에 레이블이 필요 없습니다. 입력 X를 정답 y로 삼아 자기 자신을 재구성하도록 학습하므로 비지도 학습입니다. 다만 "자기 지도 학습(Self-supervised)"으로 분류하기도 합니다.

이상 탐지에 오토인코더를 쓰는 원리는?

→ 정상 데이터만으로 학습하면 모델이 정상 패턴의 복원에 최적화됩니다. 이상 데이터는 잠재 공간에서 잘 표현되지 않아 복원 오차(Reconstruction Error)가 커집니다. 이 오차를 임계값으로 이상 여부를 판정합니다.

오토인코더와 PCA의 차이는?

→ PCA는 선형 변환만 가능하지만, 오토인코더는 비선형 활성화 함수를 통해 비선형 구조도 포착합니다. 활성화 함수 없이 선형 오토인코더를 만들면 PCA와 유사한 결과를 냅니다.

VAE(Variational Autoencoder)는 무엇이 다른가요?

→ 일반 오토인코더의 잠재 벡터 z는 고정된 점(point)이지만, VAE의 잠재 공간은 확률 분포(평균·분산)를 학습합니다. 이 덕분에 잠재 공간을 부드럽게 샘플링해 새로운 데이터를 생성할 수 있어 생성 모델로 활용됩니다.
